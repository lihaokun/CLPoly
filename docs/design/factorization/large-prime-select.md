# 根因分析：`__select_prime` 小素数导致的精度不足

**状态**: 待实现
**日期**: 2026-03-11
**关联**: `docs/design/factorization/fix-oversplit.md`（B1 根因分析，方向 5 的 B2 阻塞）
**关联**: `docs/design/factorization/univar-correctness-analysis.md`（B2 修复的正确性证明）

## 1. 问题链

B1 修复（`fix-oversplit.md`）引入方向 5（随机化求值点），但被 B2 阻塞后回退：

```
B1: over-splitting
  → 方向 5: 随机化求值点 α ∈ [-B, B]^{n-1}（降低 over-splitting 概率）
    → 大 α 代入后单变量多项式系数膨胀: |coeff(f(x,α))| = O(α^deg)
      → B2: __lll_factorize Phase 2 触发条件不完备（已修复）
        → 但 Phase 2 是后备修复，增加一次完整 Hensel 提升+重组的开销
          → 根因: __select_prime 只用小素数，导致 a_mig 高，启发式精度 a_h 不足
```

## 2. 根因分析

### 2.1 精度不足的本质

Hensel 提升的目标模数取决于两个因素：
- **Mignotte 界** $B_{\text{Mig}}(f)$：$f$ 的真因子系数的上界
- **素数 $p$ 的大小**：提升到 $p^a$ 需要 $p^a > 2 \cdot |\text{lc}(f)| \cdot B_{\text{Mig}}(f)$

精度 $a_{\text{mig}}$（达到 Mignotte 界所需的提升步数）为：

$$a_{\text{mig}} = \lceil \log_p(2 \cdot |\text{lc}(f)| \cdot B_{\text{Mig}}(f)) \rceil$$

**关键：$a_{\text{mig}}$ 与 $\log p$ 成反比。** 小素数 $p$ 使 $a_{\text{mig}}$ 线性增长。

### 2.2 启发式精度的失配

`__heuristic_starting_precision` 返回 $a_h = \min(a_{\text{mig}}, a_{h,\text{FLINT}})$，其中 FLINT 启发式：

$$a_{h,\text{FLINT}} = \biggl\lceil \frac{(2.5r + \text{min\_b}) \ln 2}{\ln p} + \frac{\ln(N+1)}{2 \ln p} \biggr\rceil$$

此公式的核心假设是：**系数不"太大"**——多项式的系数规模与次数和因子数处于同一量级。对典型的因式分解输入（小系数、小次数），此假设成立。

当 $B_{\text{Mig}}$ 因系数膨胀而变大时，$a_{\text{mig}}$ 远超 $a_{h,\text{FLINT}}$，Phase 1 精度不足。

### 2.3 B2 复现案例量化

来源：方向 5 使用 $B = 1000$ 时，crosscheck 中 $\alpha = (400+, \ldots)$ 代入产生的单变量多项式。

$f = 96000x^4 + 38464001x^3 - 61401631999x^2 - 24616960640000x - 24555520640000$

| | $p = 7$ | $p = 2^{64}-59$ |
|---|---------|-----------------|
| $\log_2 p$ | 2.8 | 64 |
| $2 \cdot |\text{lc}| \cdot B_{\text{Mig}}$ | $\approx 2^{63}$（估算） | 同左 |
| $a_{\text{mig}} = \lceil 63 / \log_2 p \rceil$ | **23** | **1** |
| $a_{h,\text{FLINT}}$ | 6 | 1 |
| $a_h = \min(a_{\text{mig}}, a_{h,\text{FLINT}})$ | 6 | 1 |
| $a_h < a_{\text{mig}}$? | **是**（$6 \ll 23$）| **否**（$1 = 1$）|
| Phase 1 充分? | 否 → 需 Phase 2 | **是** |

**结论**：大素数将 $a_{\text{mig}}$ 从 23 降到 1，消除精度失配。

### 2.4 GCD 优化的先例

P0 大素数优化（`docs/design/gcd-opt/architecture.md` §P0）已在 GCD 路径中解决了相同类型的问题：

> Brown §4.4 eq.43 推荐 $p \in [\frac{1}{2}(\beta+1),\, \beta]$。大素数使每个素数贡献 ~64 bits（vs 小素数 ~1-17 bits），CRT 循环从 ~200 次降到 ~7 次。

因式分解中的 `__select_prime` 未应用此原则——它从 $p = 2$ 开始枚举，选 $r$ 最小的素数。这是 B2 的架构性根因。

### 3.3 关键发现

1. **小系数多项式**：小素数完胜（r 更小 + DDF 便宜），大素数无优势
2. **大系数低次（deg4）**：Phase 2 开销极小（<0.05ms），不值得花大素数 DDF 代价避免
3. **大系数高次（deg10）**：大素数因 r 更优（r=6=s）且无 Phase 2 而**胜出**
4. **Phase 2 开销与素数无关时（如 p=7 deg4 只需 0.044ms），B2 fix 足够**
5. **Phase 2 开销显著时（如 p=101 deg10 需 0.400ms），大素数避免 Phase 2 有实际收益**

因此：**最优策略取决于多项式特征**，不存在单一最佳选择。

## 4. 修复方案：双模式 + 实验确定切换边界

### 4.1 两种素数选取模式

基于实验数据，实现两种独立的素数选取策略，用全局变量切换：

```cpp
enum class PrimeSelectionMode { SMALL, LARGE };
inline PrimeSelectionMode __g_prime_mode = PrimeSelectionMode::SMALL;
```

**SMALL 模式**（当前代码）：从 $p = 2$ 顺序枚举，至多 3 个合法素数取 $r$ 最小者。

**LARGE 模式**：从 $p = 2^{64} - 59$ 向下枚举（`prev_prime_64`），至多 3 个合法素数取 $r$ 最小者。

向下枚举而非随机采样的理由：
- Chebotarev 密度定理保证任意不同素数的 Frobenius 独立，无需随机化
- 确定性，可复现
- 与 GCD P0 的大素数策略风格一致（`prev_prime_64`）
- $p$ 最大化 → $a_{\text{mig}}$ 最小化

### 4.2 切换边界（待实验确定）

通过双模式对完整 `factorize()` 端到端 benchmark，确定切换条件。候选维度：
- 系数规模：$\max|c_i|$ 或 $\|f\|_\infty$
- 次数：$\deg f$
- 项数：$|f|$
- 综合指标：$a_{\text{mig}}$ 的估算值（可在选素前用 Mignotte 界预估）

### 4.3 兼容性

DDF/EDF 对 64-bit 大素数兼容（M0.4 已修复 Zp 大素数安全）：

| 组件 | 兼容性 | 说明 |
|------|--------|------|
| `Zp` 模算术 | ✓ | Barrett 归一化（M0.4） |
| `__ddf_Zp` | ✓ | `powmod(h, ZZ(p), f*)`，64 次平方 |
| `__edf_Zp` | ✓ | 指数 $(p^d-1)/2$ 为 `ZZ`，任意大 |
| `polynomial_GCD(Zp)` | ✓ | HGCD/Euclid 均支持大素数 |

## 5. 与 B2 修复的关系

B2 修复（`univar-correctness-analysis.md` 定理 3.5）和大素数选取是**互补**的两层防御：

| 层 | 机制 | 作用 |
|----|------|------|
| **B2 fix**（已实现）| Phase 2 触发条件 `result.size() < r && a_h < a_mig` | 正确性保证：精度不足时自动升级到 Mignotte |
| **大素数模式**（本方案）| 大素数使 $a_{\text{mig}} \le 2$，$a_h \ge a_{\text{mig}}$ | 性能保证：避免触发 Phase 2 |

B2 fix 是必要的安全网（任何素数下都保证正确），大素数是性能优化（减少 Phase 2 触发频率）。两者无论哪种模式都保持正确性。

## 6. 与方向 5 的关系

本方案完成后，方向 5（`fix-oversplit.md` §4）的两个阻塞因素均已解除：

| 阻塞因素 | 状态 | 解除方式 |
|----------|------|----------|
| B2: Phase 2 触发条件不完备 | **已修复** | `result.size() < r && a_h < a_mig` |
| 大系数触发 Phase 2 的性能开销 | **本方案解决** | 大素数模式使 Phase 2 几乎不触发 |

方向 5 可安全恢复，使用 $B = \max(100, 2 \cdot \text{tdeg\_sum})$ 或更大。

### 7.2 切换边界实验方案

构造系统化用例矩阵，对每个用例在 SMALL/LARGE 两种模式下跑完整 `factorize()` 端到端对比：

**维度 1：系数规模**（固定 deg=10, 4 factors）
- $\|f\|_\infty \approx 10^1, 10^3, 10^5, 10^7, 10^9, 10^{12}, 10^{15}$
- 通过调整因子常数项构造：$(x + \alpha)(x - \alpha)(x + 1)(3x + 7)$，$\alpha = 1, 30, 300, 3000, \ldots$

**维度 2：次数**（固定大系数 $\alpha = 500$）
- deg = 4, 8, 12, 16, 20, 25, 30
- 通过累乘随机二次因子构造

**维度 3：因子数**（固定 deg≈20, 大系数）
- 2 因子（2 个 deg10）、4 因子（4 个 deg5）、10 因子（10 个 deg2）、20 因子（20 个线性）

对每个用例记录两种模式的：
- 端到端 `factorize()` 时间
- 选中的 $r$、$a_h$、$a_{\text{mig}}$
- Phase 2 是否触发

预期找到切换曲面在（系数规模, 次数）平面上的近似边界。
