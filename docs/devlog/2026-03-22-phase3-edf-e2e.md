## 为什么做

Zp 因式分解管线 L2 完成（SQF + DDF + EDF），自然接入 Pipeline 框架得到端到端定理。EDF 是管线最后一块，数学上最简单（无循环不变量，只有"分裂→递归"）。

## 关键决策

1. **edfSplit inline**：不拆为独立函数。`if hg : ...` 的度数 bound 直接在 `decreasing_by` context 中可用，避免跨函数传递证明。

2. **去随机化**：`splits : List` 替代 `while(true)` + RNG。正确性是结构性的（定理 A/B 对任意 splits 无条件成立）。

3. **组合度量** `f.natDegree * (splits.length + 1) + splits.length`：单个自然数编码 lex 对 `(natDegree, splits.length)`。`none` 分支 splits.length 减 1，`some` 分支 natDegree 减小。用 `nlinarith` 证递减。

4. **g = 1 边界**：Pipeline 的 `hedf` 可能对 DDF 输出的 unit（度 0）调用。解法：包装 edf，natDegree = 0 时返回 `[]`（空列表满足 EDFCorrect for g = 1）。

## 遇到的问题

| 问题 | 解法 |
|------|------|
| `decreasing_by` 看不到 `let h := ...` 的度数 bound | 去掉 `let h`，直接用 `normalize (f /ₘ g)` |
| `Nat.strongRecOn` 的 `none` 分支需 `n < n`（不可能） | 改用组合度量，单个 Nat 编码 lex 对 |
| `nlinarith` 证 `a * (S+1) + S < b * (S+2) + S+1` when `a < b` | 提供 `hg_deg_lt` 作为 nlinarith hint |
| `Monic.leadingCoeff` 与 `coeff 0` 不匹配（natDegree = 0） | 手动 `rw [leadingCoeff, hg_deg]` 统一 |
| `List.not_mem_nil` 类型变化 | 用 `simp at hq` 替代 |
