# CAD Cell 管理结构 调研报告

## 1. 项目现有基础设施

### 1.1 投影算子（已完成）

`clpoly/cad.hh` 实现了完整的投影阶段：
- `__project_full(polys, vars, method)` — 逐层计算投影集，返回 `allprojs[k]` = 第 k 层多项式集合 P_k
- 投影算子：McCallum 和 Lazard 均已实现
- `allprojs[k]` 中所有多项式包含第 k+1 个变量，不含更高维变量

### 1.2 实根隔离（已完成）

`clpoly/realroot.hh` 中的 `uroot` 类：
```cpp
class uroot {
    int _is_inf;           // -1/0/1
    upolynomial_ZZ _poly;  // 定义该根的一元多项式
    QQ _left, _right;      // 隔离区间
};
```
`realroot(polys)` 对一组多项式做无平方基后，用 Uspensky 算法输出 `vector<uroot>`。

### 1.3 采样函数（已完成）

`sample_open_intervals(polys)` — 对给定多项式的实根隔离结果，在每个开区间内各取一个有理采样点。

### 1.4 Cell 草稿（已废弃）

`clpoly/cell.hh` 全部注释掉，其设计为：
```
root<Tc,comp>       存储 polynomial_（按值）+ variable + root_index
root_interval       存储 1~2 个 root（Section / Sector）
cell                存储 vector<root_interval>，每维一个
```

**核心问题**：每个 `root` 持有一份完整 `polynomial_` 副本。若有 M 个 cell 共享同一投影层多项式集 P_k，每个多项式被存储 O(M) 次。

## 2. CAD 算法结构分析

### 2.1 数学结构

CAD 分解 R^n 为有限个半代数集（胞腔）。令变量序为 x_1 > x_2 > ... > x_n（字典序）：

- **第 1 层**：P_1 ⊆ Z[x_1] 的实根把 R 分为有限个区间（Section 在根上，Sector 在根之间）
- **第 k 层**：给定第 (k-1) 维 cell 的采样点 (α_1,...,α_{k-1})，P_k(α_1,...,α_{k-1}, x_k) 的实根把对应 x_k 轴分为区间

**关键性质**：
1. **P_k 对整层共享（初版）**：标准投影后，层 k 所有 cell 的边界多项式都来自同一投影集 P_k
2. **提升阶段可引入新多项式**：例如处理 nullification（多项式在采样点处恒为零）等情形时，需向当前层追加新多项式，`level_polys[k]` 须支持**追加**而不破坏已有索引
3. **树形层次**：k 维 cell 的子节点是所有以它为父节点的 (k+1) 维 cell，天然形成树
4. **叶节点 = 完整 cell**：n 维叶节点对应 R^n 中一个完整胞腔

### 2.2 采样点的代数表示

**当前**：每个节点存一个有理采样点 `QQ`（Sector 取开区间内的有理数，Section 取根的有理近似）。

**完整 CAD 所需**：采样点一般是代数数，需要精确表示。多维代数数的标准方式是
**正规三角列 + 有理隔离区间列**：
- 正规三角列 {f_1(x_1), f_2(x_1,x_2), ..., f_k(x_1,...,x_k)}
- 有理隔离区间列 {[a_1,b_1], ..., [a_k,b_k]}
- 语义：α_i 是 f_i(α_1,...,α_{i-1}, x_i) 在 [a_i,b_i] 内的唯一实根

**Section 路径 ≠ 三角列（直接）**

树路径上 Section 节点的多项式来自 CAD 投影集 P_k，这些多项式未必构成正规三角列：
- 投影多项式可能非不可约，含额外因子
- 在采样点处可能退化（nullification：某多项式恒为零）
- 相邻层的多项式之间不一定满足正规三角列的 pseudo-remainder 条件

因此，若要精确表示代数采样点，需从 Section 路径的信息出发，**归约计算出正规三角列**，再配合有理隔离区间得到标准表示。

**分层设计策略**：
- **近期**：Sector 节点存 `QQ sample_point`（有理数），Section 节点存 `(poly_idx, root_idx)` 索引
- **远期**：实现代数数类型（正规三角列 + 有理区间）后，为每个节点提供按需计算/缓存精确代数采样点的接口，无需改变树节点的核心存储

### 2.3 数据流

```
输入多项式 F
  ↓ __project_full()
allprojs[0] = P_1 ⊆ Z[x_1]          ← level_polys[0]（可追加）
allprojs[1] = P_2 ⊆ Z[x_1,x_2]      ← level_polys[1]（可追加）
  ...
allprojs[n-2] = P_{n-1}              ← level_polys[n-2]（可追加）
  ↓ realroot() + 根编号
level-1 节点（Section/Sector，存 poly_idx + root_idx 索引）
  ↓ 对每个采样点代入 level_polys[1]，再 realroot()
level-2 节点
  ...
叶节点 = 完整 cell
```

## 3. 方案对比

### 方案 A：`shared_ptr` 共享多项式

`root` 中将 `polynomial_` 改为 `shared_ptr<const polynomial_>`。

| 维度 | 评价 |
|------|------|
| 存储冗余 | 部分消除（引用计数本身有开销） |
| 接口改动 | 最小 |
| 所有权语义 | 分散，不清晰 |
| 与算法结构的契合度 | 低（cell 不知道自己属于哪一层） |

**结论**：治标不治本，不推荐。

### 方案 B：索引式 root

cell 不存多项式，只存 `(poly_idx, root_idx)` 指向外部投影集。

| 维度 | 评价 |
|------|------|
| 存储冗余 | 消除 |
| 接口改动 | 中等（需要外部 context） |
| 所有权语义 | 集中（投影集统一管理） |
| 与算法结构的契合度 | 高 |
| 缺点 | cell 离开管理类无法独立解释 |

**结论**：适合只需存储最终 cell 集合的场景。

### 方案 C：树形结构 + 索引式 root（推荐）

以树节点组织 cell，多项式集 P_k 存在树的层级元数据中，节点只存索引和采样点。

| 维度 | 评价 |
|------|------|
| 存储冗余 | 消除 |
| 与算法结构的契合度 | 最高（提升过程即树的逐节点展开） |
| 管理能力 | 强（天然支持按层遍历、叶节点枚举）|
| 复杂度 | 适中 |

**结论**：推荐。

## 4. 推荐方案

**方案 C：`cad_tree` 树形管理结构**

核心设计原则：
1. **投影集按层集中存储，且可追加**：`level_polys[k]` 存 P_{k+1}，所有节点共享，不复制；支持提升阶段追加新多项式，已有索引不失效
2. **节点轻量化**：节点用 `cad_root`（`poly_idx, root_idx` 或 ±∞）描述边界——Section 存 1 个 `cad_root`，Sector 存 2 个——加可选有理采样点
3. **分层存储**：`levels[k]` 存第 k 层所有节点，节点间通过层内下标引用。单个 cell 表示为 `vector<size_t> path`（每层一个下标），轻量自然
4. **数据结构与算法分离**：`cad_tree` 只做数据管理，提升算法（`__open_cad`、`__full_cad`）为外部自由函数，与 `__project_full`/`realroot` 风格一致
5. **叶节点 = 完整 cell**：遍历叶节点即遍历所有胞腔
6. **代数采样点按需计算**：Section 节点的 `cad_root` 索引提供了计算正规三角列所需的原始信息，精确代数采样点由归约计算得出，不预先存储

分层存储的选择理由：
- 提升算法逐层处理，分层存储下同层节点连续，缓存友好
- 单个 cell / 部分 cell 集合的表示自然（每层一个下标 / 每层一个有效下标集合）
- 不需要虚根节点，结构更简洁

与现有代码的衔接：
- `__project_full()` 的输出直接成为 `level_polys` 的初始内容
- `realroot()` 用于逐层展开节点（返回 `uroot`，含隔离区间和根编号）
- `sample_open_intervals()` 用于 Sector 节点取有理采样点
- `uroot` 已有隔离区间信息，可用于构造节点的边界描述

**未来代数数扩展路径**：
- 精确代数采样点的表示：正规三角列 + 有理隔离区间列（二者合一，唯一确定代数点）
- Section 节点已有 `(poly_idx, root_idx)` 链，加上隔离区间即可触发归约计算得到正规三角列
- Sector 节点的有理采样点未来升级为：归约后的正规三角列 + 有理区间（统一表示格式）
- 代数数类型实现后，通过节点上的接口按需获取，核心树结构不变

## 5. 调研结论

| 问题 | 结论 |
|------|------|
| 当前草稿冗余在哪？ | `root` 按值存储 `polynomial_`，M 个 cell 重复存储 O(M) 份 |
| 如何消除？ | 投影集按层统一存储，cell/节点改为索引引用 |
| 最自然的组织方式？ | 树形结构 + 分层存储（`levels[k]` 存第 k 层所有节点），与 CAD 提升算法的逐层展开过程完全对应 |
| 节点如何描述边界？ | 统一用 `cad_root`（`poly_idx, root_idx` 或 ±∞），Section 存 1 个，Sector 存 2 个 |
| 单个 cell 如何表示？ | `vector<size_t> path`，path[k] = levels[k] 中的下标 |
| 与现有代码如何衔接？ | `__project_full()` 的输出直接用作 `level_polys` |
| 提升阶段引入新多项式？ | `level_polys[k]` 追加即可，索引式引用不受影响 |
| 算法与数据结构？ | 分离：`cad_tree` 管理数据，`__open_cad` 等为外部自由函数 |
| 代数采样点？ | 表示为"正规三角列 + 有理隔离区间列"；Section 路径提供原始信息但需归约正规化；近期 Sector 节点存 `QQ`，远期统一为正规三角列表示 |
